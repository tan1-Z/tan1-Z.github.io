从写作到发布：Hexo 博客全流程实战

发表于2026-01-06|更新于2026-01-06|实战笔记
|浏览量:

这篇记录我自己的博客工作流：从写一篇文章，到本地预览，再推送到 GitHub Pages。命令行一步步列清楚，照着做就能跑通。

1. 新建与书写

  1. 新建文章（自动生成 Front Matter）：

    默认会在 source/_posts/ 下生成一个同名 Markdown。建议在 Front Matter 填上 tags、categories、date 方便归档。

  2. 写作时的常见小技巧：

  • 图片放 source/img/，文章里引用 /img/xxx.jpg。
  • 代码块用三反引号围住，注明语言更易高亮。
  • 草稿模式可先放 source/_drafts/，预览时加 hexo server --draft。

3. 部署到 GitHub Pages

确保 _config.yml 的 deploy.repo 指向你的 Pages 仓库（我的是 https://github.com/tan1-Z/tan1.github.io.git，分支 gh-pages）。

4. 常见问题排查

  • Cannot GET /：通常是 source/_posts 为空或生成失败。hexo clean && hexo g 后再看 public/index.html 是否存在。
  • 端口被占用：hexo server -p 4002 换个端口。
  • 部署失败 Failed to connect to github.com：检查网络/代理；确认 deploy.repo 写对；必要时重试。
  • 头像/封面不更新：改完 _config.butterfly.yml 后一定 hexo clean && hexo g，浏览器强制刷新或清缓存。
